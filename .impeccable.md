# .impeccable.md — Glyph UI Design Context

## Design Context

### Users
**Finance teams and accountants** using an internal dashboard (Ledger) to manage company money day-to-day. They are power users: experienced with data-dense UIs, value precision and speed over decoration, and need to trust what they see immediately. Context is professional — typically desktop, sustained work sessions, moderate-to-high information density expected.

### Brand Personality
**Efficient. Precise. Trustworthy.**

The product should feel like a well-engineered tool, not a marketing page. Every pixel earns its place. There is no room for decoration that doesn't carry meaning.

### Aesthetic Direction
- **Visual tone:** Utilitarian and refined — dense data without visual noise
- **Design system:** GlyphUI exclusively — use existing tokens (Inter font, purple-600 accent, slate neutrals, spacing scale, radius tokens) and existing components (GlyphDataTable, GlyphCard, GlyphBadge, GlyphTextField, GlyphButton, GlyphDropdown, GlyphAppBar, GlyphPagination, etc.)
- **Theme:** Light only — white surfaces, slate neutrals, purple accent for interactive elements
- **References:** Stripe Dashboard, Brex, Mercury — confident type hierarchy, tight layouts, deliberate color
- **Anti-references:** Glassmorphism, gradient backgrounds, oversized decorative icons, hero-number layouts with glows

### Design Principles
1. **Use the system** — Every color, spacing value, radius, and text style comes from GlyphUI tokens. No raw values.
2. **Density is clarity** — Finance users can read more when the layout is disciplined. Generous padding is not the same as good design.
3. **Hierarchy through type** — Rely on GlyphTextStyles scale (title → label) to create structure. Color is secondary to size and weight.
4. **Components compose** — Use existing GlyphUI components as building blocks. New screens are compositions, not new inventions.
5. **Interaction is minimal** — State changes (hover, press, loading) exist; decoration does not.
